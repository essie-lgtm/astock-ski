## 怎么跑
浏览器对 `file://` 下的 `fetch()` 有跨域拦截，必须起一个本地静态服务器：

```bash
cd astock-snowboard
python3 -m http.server 8000
# 然后浏览器打开 http://localhost:8000/
```

想玩别的票：`http://localhost:8000/?code=600519`（需先生成对应 JSON）。

## 操作（Ski Safari 式一键控制）
- 滑行**自动进行**，不用管前进（下坡加速、上坡减速、上坡不卡死）。
- **点按**空格 / 屏幕 = 即时跳跃（跟手）。
- 腾空时**按住** = 后空翻；**松手** = 停转、自然平衡回正，准备落地。
- 板面贴坡面落地才稳；歪了会摔一下（约 1 秒翻滚 + 掉速），但会**自动爬起继续滑**，不会结束游戏。
- 重开：按 `R` 或结束页的「再滑一次」按钮（空格不再用于重开）。

## 地形是怎么来的（看懂"平的一段"和"金墙")
- **纵轴用对数价格**：这样任何价位的"涨跌百分比"都看得见。低价区(如 ¥18 那段)的小波动在普通线性图上几乎是平的，对数后变成有起伏的缓坡。所以"长时间平滑"其实是那段时间波动很小（外加去趋势后更接近水平），股价在变但变得小。
- **去趋势**：妖股整体 +516%，若直接当坡度就是一条没完没了的上坡。去掉整体大斜坡，赛道在水平线附近有起有伏，показ的是"波动"而不是"净涨跌"。整体盈亏仍在 HUD 看。
- **大事件 = 持续的台阶，不是对称小包**：涨停那天纵向落差被放大且**一直保留**（涨上去就待在高处），所以你冲上金色发射墙后是**起跳腾空**落到后面的地形，而不是"滑下小山包却没有跌幅"。真正的下坡只发生在真·下跌日（绿色悬崖）。

## 地形可骑性（已做）
- 收盘价不再用直线段硬连（那样每天一个尖角）。先对**普通日**做三角权重均线压抖动，再用**单调三次样条(Fritsch–Carlson)** 在交易日之间补 `TERRAIN_SUBSTEPS` 个过渡点重采样 → 坡度连续、无尖锐拐角。
- **大事件保留**：`limit_flag≠0` 或 `|pct|≥BIG_EVENT_PCT` 的日子不参与平滑，仍是陡崖/发射墙——该抛飞就抛飞。即"小波动磨平成缓坡，大波动保留成大坡"。
- **滑板贴地**：滑行时顺着地形轮廓走，只有凸度≥`LAUNCH_CONVEXITY_MIN` 且速度够快的真陡坡才离地，小起伏一律贴地滑过；且没在做特技(几乎没转)的落地永远算干净，地形颠簸绝不害你摔。

## 调手感
所有可调常量都在 **`js/config.js`**，改完刷新即可；也可在浏览器控制台直接改 `window.CONFIG.xxx` 实时试。
地形相关：`TERRAIN_SUBSTEPS`(平滑细分)、`TERRAIN_SMOOTH_WINDOW`(均线半窗)、`OHLC_AMP`(日内高低影线起伏夸张,0=退回收盘单线)、`BIG_EVENT_PCT`(大事件阈值)、`LAUNCH_CONVEXITY_MIN`/`LAUNCH_SENSITIVITY`(起跳难易)。
速度系统(自限速)：`CRUISE_SPEED`(平地巡航)、`SLOPE_ACCEL_K`(坡度增速,已调小)、`DRAG_QUAD`(二次阻力,越大终端速度越低)、`MAX_SPEED`(硬上限,含腾空)。默认偏保守(巡航≈58km/h，仅陡崖短暂冲到≈126km/h再被拉回)。
相机：`CAM_LOOKAHEAD`(前瞻秒数)、`CAM_MIN_ZOOM`(高速拉远程度)。
熊市雪崩：`AVALANCHE_ENABLED`(开关)、`AVALANCHE_BASE_FRAC`/`AVALANCHE_SURGE_FRAC`(前压速度,巡航倍数)、`AVALANCHE_FAR_GAP`/`AVALANCHE_NEAR_GAP`/`DANGER_GAP_CURVE`(牵引绳:跌幅越大拽越近)、`AVALANCHE_WARN_DIST`(预警距离)、`CAUGHT_GRACE_FRAMES`(被抓容错帧)。想更凶就调大 `SURGE_FRAC`、调小 `NEAR_GAP`。
单局时长：`SEGMENT_WIDTH`(每日像素宽度，调大=更长更缓)。当前 150 ≈ 1.8min。
纵向/趋势：`LOG_TO_PX`(对数价格→像素,越大越陡)、`DETREND_STRENGTH`(去趋势,默认 0.5)。
  - `0.5`=折中默认：涨票净上坡但仍有速度起跳、跌票净下坡，两类都能玩。
  - `0`=忠实保留趋势：**下跌票=一路俯冲下坡**（套牢主题最强），但强涨票会变成一路爬坡。
  - `1`=完全拉平：只看波动、不看净涨跌，最好骑但没有"大趋势"。
大事件夸张：`BIG_EVENT_GAIN`(大事件纵向放大倍数,越大墙越高)、`BIG_EVENT_PCT`(算大事件的阈值)。
跳水段(连续阴跌)：`DIVE_DD_PCT`(从近峰回撤多少算跳水)、`DIVE_FALL_WIN`(同时要求近 N 日仍在跌)、`DIVE_GAIN`(段内下跌日陡化倍数,越大俯冲越猛)、`DIVE_MIN_RUN`(够长才打标签)、`DETREND_DOWN_STRENGTH`(净下跌票的去趋势强度,越小越保留俯冲长坡)。
屏幕反馈：`FEEDBACK_PCT`(触发阈值)、`FEEDBACK_PUNCH`(镜头猛推)、`FEEDBACK_SHAKE`(抖动)、`FEEDBACK_FLASH`(闪色)、`FEEDBACK_DECAY`(消失快慢)。

## 快速换票
- 网址参数：`http://localhost:8000/?code=000687`（代码需先用 `fetch_stocks.py` 生成对应 JSON）。
- 开始页有输入框 + 下拉（下拉项来自 `data/index.json`），输入代码点「换票」即可。
重力、坡度增速系数、地形纵向放大、起跳弹跳、空中转速、落地容错角等都在里面，
雪崩相关常量（基础速度、浪涌系数 k、danger 平滑窗口、预警距离、被抓容错帧）也已占位，Step 3 启用。

调试钩子（控制台）：`__GS()` 看骑手/分数状态，`__hold(true/false)` 强制按住，`__warp(进度0~1, 速度)` 瞬移测试。
